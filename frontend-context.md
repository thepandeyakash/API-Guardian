# API Guardian Frontend

You are a senior frontend engineer.

The backend for API Guardian is already complete.

IMPORTANT RULES:

* First analyze the backend implementation.
* Infer API request and response shapes from the backend code.
* Never invent API endpoints.
* Never create mock APIs.
* Never create fake response objects.
* Reuse existing backend routes exactly.

Tech stack:

* React
* TypeScript
* Vite
* Tailwind CSS v4
* shadcn/ui
* TanStack Query
* Zustand
* React Router
* Recharts

Design inspiration:

* Datadog
* Grafana
* Sentry
* Vercel Dashboard

Design requirements:

* Dark theme
* Professional SaaS dashboard
* Minimalistic
* Data dense
* Responsive

Architecture requirements:

1. Create API service functions.
2. Create React Query hooks.
3. Create Zustand stores when necessary.
4. Create reusable components.
5. Create page UI.
6. Connect the backend APIs.
7. Handle loading, error, and empty states.

Do not build the entire frontend at once.
Build page by page.
